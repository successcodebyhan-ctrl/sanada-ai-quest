# 真田幸村的AI征途 - 開發指令文件

## 專案概述
一款2D像素風橫向卷軸知識學習遊戲，玩家扮演真田幸村，穿越三個歷史時代，以答題方式與歷史名將對戰，最終集齊三顆寶石返回故鄉。

## 技術規格
- **平台**：網頁版（HTML + CSS + JavaScript）
- **目標瀏覽器**：手機與電腦的 Google Chrome
- **螢幕支援**：需要偵測並支援垂直（直立）與橫向（橫放）螢幕切換
- **語言**：全繁體中文
- **畫面風格**：2D 像素風（Pixel Art）卷軸

---

## 遊戲架構

### 三大主題與時代對應
| 主題 | 時代 | 大魔王 | 寶石 |
|------|------|--------|------|
| Claude AI | 三國時期 | 曹操 | 曹操的翡翠 |
| Claude Cowork | 騎士時期 | 亞瑟王 | 亞瑟王的紅寶石 |
| Claude Code | 江戶時期 | 德川家康 | 德川家康的黃玉 |

### 關卡結構
- 每個時代各有 **5 個關卡**（含大魔王關）
- 每關有 **5～10 道題目**（詳見 questions.json）
- 大魔王關固定 10 題
- 所有題目資料存於 **questions.json**，遊戲從此檔案讀取

## 畫面與場景

### 1. 開場動畫（30秒）
- 圖片交替 + 文字說明的靜態動畫
- 共 6 個畫面，每個畫面 5 秒
- 畫面內容：
  1. 黑色背景 + 戰場像素圖 → 文字：「慶長二十年，大阪夏之戰。」
  2. 真田幸村倒下的像素圖 → 文字：「真田幸村，最後一戰，力竭而亡。」
  3. 光芒從黑暗中透出 → 文字：「然而，翌日清晨，他睜開了雙眼。」
  4. 領地全景圖，傳送門在遠方發光 → 文字：「一個陌生的領地，一扇神秘的石門，等待著他。」
  5. 傳送門特寫，三個寶石缺口清晰可見 → 文字：「門上有三個缺口，據說集齊三顆寶石，便能回到故鄉。」
  6. 真田幸村握刀站立 → 文字：「真田幸村的新征途，現在開始。」→ 淡入主畫面

### 2. 真田幸村的領地（主畫面）

**畫面配置**：
- 2D 橫向場景
- 靠近建築時，建築上方顯示像素風文字的建築名稱

**建築清單**：

#### 真田屋（可進入）
- 風格：江戶「武家敷屋」，屋簷有「六文錢」家徽
- 進入後有：榻榻米、床墊、盔甲座、武士刀座
- 進出有動畫效果

#### 村正鐵匠鋪（不可進入）
- 開放式小舖，無門
- 鐵匠在門口製作武士刀
- 招牌：「村正」二字
- 點擊聊天 → 鐵匠說：「幸村，你的使命配得上村正家的刀。」
- 無其他功能

#### 雜貨舖（可進入）
- 江戶時期風格
- 老闆站在櫃檯後
- 靠近老闆 → 開啟商店介面
- **商品清單**：
  | 商品 | 回血量 | 售價 |
  |------|--------|------|
  | 蕎麥麵 | +15 HP | 30 銅幣 |
  | 烤魚 | +30 HP | 60 銅幣 |
  | 天婦羅蓋飯 | +50 HP | 100 銅幣 |

#### 軍隊扎營處（不可進入，景觀用）
- 有武士帳篷和火堆
- 幾名武士坐在火堆旁並隨機走動
- 打敗各時代大魔王後，在此插入對應旗幟：
  - 三國時代旗幟（打敗曹操後）
  - 騎士時代旗幟（打敗亞瑟王後）
  - 江戶時代旗幟（打敗德川家康後）

#### 石製傳送門（互動式建築）
- 石頭製門框，門框內有半透明紫色漩渦
- 靠近後開啟選單：
  1. 進入三國時代（Claude AI）
  2. 進入騎士時期（Claude Cowork）
  3. 進入江戶時期（Claude Code）
  4. 放入寶石（已集齊相應寶石時才可選）
- 放入一顆寶石時：漩渦變金色，然後恢復紫色
- 集齊三顆寶石 → 觸發結局動畫

### 3. 時代場景（選擇時代後的畫面）

**通用規則**：
- 畫面左邊：石製傳送門（互動：只有「回到領地」選項）
- 畫面右邊：對應時代的城堡/領地
- 靠近右邊建築 → 開啟關卡選單
- 破完第 N 關才能進入第 N+1 關

#### 三國時期（Claude AI）
- 右側：三國風格城堡
- 元素：軍旗、隨機走動的三國兵將
- 背景音樂：古箏、琵琶風格

#### 騎士時期（Claude Cowork）
- 右側：中世紀貴族城堡
- 元素：弓箭手、穿著中世紀盔甲走動或騎馬的騎士
- 背景音樂：管風琴、魯特琴風格

#### 江戶時期（Claude Code）
- 右側：以岡崎城為視覺設計的建築
- 元素：武士、馬
- 背景音樂：尺八、太鼓風格

---

### 4. 關卡選單
- 顯示 5 個關卡
- 已解鎖：顯示關卡名稱、敵將名稱、難度星星、最高星評（1-3顆星）
- 未解鎖：顯示鎖頭圖示

---

### 5. 關卡進行流程

#### 第一階段：知識教學
- 一名隨從武士出現，用**聊天對話框**教授本關學習主題
- 知識內容根據 questions.json 中的 stageName 對應
- 玩家點擊「繼續」完成閱讀後，進入對戰

#### 第二階段：對戰畫面

**佈局（橫屏）**：
```
[左上：答題數/總題數]          [右上：撤退按鈕]
[敵人名字]                     [真田幸村]
[敵人血條 ████████]            [血條 ████████ 100HP]
[敵人像素角色]                 [真田幸村像素角色]
         [題目與選項區域（中央下半部）]
                               [右下：道具欄]
```

**血量機制**：
- 每關開始時血量自動補滿至 **100 HP**
- 每題扣血量 = 100 ÷ 總題數
  - 5題關卡：答錯扣 20 HP
  - 7題關卡：答錯扣約 14 HP
  - 10題關卡：答錯扣 10 HP
- 答對 → 敵人扣等量血量
- 答錯 → 我方扣等量血量

**勝負判定**：
- 血量歸零 → 失敗 → 返回領地（不扣任何道具）
- 通過 → 顯示破關畫面

**星評系統**（依剩餘血量）：
| 剩餘血量 | 星評 | 意義 |
|---------|------|------|
| 81-100 HP | ⭐⭐⭐ | 優秀 |
| 41-80 HP | ⭐⭐ | 良好，建議複習 |
| 1-40 HP | ⭐ | 勉強過關，強烈建議複習 |

**道具系統**：
- 道具可在答題**過程中隨時使用**（不需等到關卡結束）
- 道具欄顯示在畫面右下角
- 使用道具 → 立即回復對應血量

**撤退**：
- 點擊右上角撤退按鈕 → 立即返回領地
- 該關進度**不保留**，下次重新作答

**題目類型**：
1. **選擇題**：4個選項，點擊選擇
2. **填空題**：提供4個詞語讓玩家選填，點擊填入空白處

---

#### 第三階段：破關畫面
- 顯示：「關卡完成！」
- 顯示：星評（1-3顆星）
- 顯示：掉落寶物
- 選單：「繼續下一關」或「回到領地」

## 存檔系統
- 使用瀏覽器 **localStorage** 儲存遊戲進度
- 儲存內容：
  - 各關卡解鎖狀態
  - 各關卡最高星評
  - 當前銅幣數量
  - 道具背包內容
  - 已獲得的寶石清單
  - 軍隊扎營處旗幟狀態

---

## 題目資料格式（questions.json）

所有題目存於 questions.json，格式如下：
```json
{
  "id": "ca-1-1",
  "type": "選擇題" 或 "填空題",
  "question": "題目文字",
  "options": ["選項A", "選項B", "選項C", "選項D"],
  "answer": "正確答案",
  "explanation": "答案解釋（破關後或答錯後顯示）"
}
```

## 響應式設計要求

### 直立螢幕（手機豎拿）
- 對戰畫面改為上下佈局
- 角色縮小，題目區域佔據較大空間
- 按鈕放大，方便手指點擊

### 橫向螢幕（手機橫拿 / 電腦）
- 使用標準橫向佈局（如上述佈局說明）
- 最佳化電腦滑鼠操作體驗

### 螢幕偵測
- 自動偵測螢幕方向並切換對應佈局
- 旋轉時平滑過渡，不重置遊戲狀態

## 開發順序建議

1. **基礎架構**：HTML 結構、CSS 像素風設計系統、JavaScript 框架
2. **資料讀取**：載入 questions.json，建立資料管理系統
3. **領地主畫面**：場景繪製、建築互動、角色移動
4. **雜貨舖系統**：商店介面、購買功能、背包管理
5. **傳送門系統**：時代選擇介面、場景切換動畫
6. **時代場景**：三個時代的場景繪製
7. **關卡選單**：關卡解鎖邏輯、星評顯示
8. **教學對話系統**：隨從武士、對話框
9. **對戰系統**：血條、題目顯示、答題判定
10. **道具系統**：背包、使用道具、回血效果
11. **破關流程**：星評計算、寶物掉落、進度儲存
12. **開場動畫**：圖片交替、文字動畫
13. **音樂系統**：各場景背景音樂
14. **響應式優化**：橫直屏切換
15. **測試與修正**

---

## 注意事項
- 所有文字使用**繁體中文**
- 像素風格要一致，建議使用 CSS 的 `image-rendering: pixelated`
- 答題後顯示解釋說明（explanation 欄位的內容）
- 保持遊戲流暢，避免卡頓
